John Christy's Notes on Intro to Programming

Lesson 1 - Basics of the Web
Hyper Text Markup Language
Better known as HTML, this is the main document type of the web. It can contain Text content, markups, and references
The Four Basic Parts of The Web
1. Browsers - This is a program run on your local computer. It connects to remote sources and displays files from remote locations
2. The Internet - This is the magical cloud of wires and connections that gets your browser to the servers.
3. Markups - these modify the way text appears on the page.
4. Servers - Servers are computers optimized to serve files. The browser is showing files that are hosted on a given server.
Markup Types
Markups can be inline like <br>, or block like <p>.
Inline markups modify an eliment in the HTML code
Block markups creates a box with a line break. Nothing besides what is contained in the block markup can be displayed on the same line without some other conditional formatting like a flexbox.
So a <p> (paragraph) would be contained inside a small box, and a piece of text that is made bold is inline.
Something to Remember - Computers are Stupid
Remember that computers are stupid and can not make best guesses at what we want. Exact instructions are key!
Lesson 2 - Document Structure
HTML Files
HTML Files are the structure of a page, and should be considered a language. The HTML file will include all of the text, links, images, etc. It is possible to do all of your styling in the HTML file, but the redundant code would get infuriating
A good way to think about HTML Files is the structure of a building. It is the floor and walls that you can later paint.
CSS
CSS adds style to the HTML, and should also be considered a language. The CSS file allows you to create general style elements so that you can utilize them in the HTML code. Creating the elements in the CSS file allow you to use a single element line instead of typing all of the attributes of each "box."
For a good reference, check out the W3Schools.com website for a great listing of CSS properties.
DOM
DOM stands for the Document Object Model, but can also be referred to as the Tree Structure. The structure refers to the way the browser creates and nests nodes in a tree like manner
Something to Remember - Everything is a Rectangle!
Everything appearing in the browser should be considered as in a "box." These boxes are styles by the CSS or markups in the HTML file.
Lesson 3 - Adding Style and Structure
Three Ways to Include Style
Add all style properties to the Header section of the HTML file.
Add a link to a CSS stylesheet in the header.
Write all style changes inline with the HTML and repeat and repeat and repeat...
The Box Model
The Box model is a way to visualize the space any given object will take up. It is made up of 4 parts:
The Content is the text, image, or other defined object at the center of the box model
The Padding is the area between the content and the border.
The Border is the definable line between the padding and the margin.
The Margin is the transparent space on the outside of the border.
Object Positioning
Positioning objects on the page is the most complicated part of coding for the web.
One tool to help position boxes side by side is called Flex
Adding display:flex to the css for a DIV will put two boxes next to each other. This is necessary because by default, each DIV is a block element and takes the entire width of its parent element.
For example, the two lesson note boxes above are in a DIV with the display:flex feature and appear side by side. The box these notes are in does not have flex, and takes the entire width of the body element.

Something to Remember - Code, Test, then refine
It is important to create a page in order. The most efficient way to code is in the following order:
1. Look for natural boxes in the page you are trying to code - These are where you should anticipate DIV's
2. Look for repeated styles in the page - This is where you will create elements in CSS to reduce repetition in the HTML code
3. Write HTML - Write all of the site code, inserting all of the proper Divs that you recognized earlier
4. Apply Styles - Create the CSS to start painting the walls of the HTML building.
5. Refine - Add spans, manipulate the margins, padding, etc. Do what it takes to make the site look good.
